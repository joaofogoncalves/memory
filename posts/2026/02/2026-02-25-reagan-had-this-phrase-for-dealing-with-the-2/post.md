# February 25, 2026

Reagan had this phrase for dealing with the Soviets.
 Trust but verify.

Didn't expect it to become my entire workflow with coding agents.

The trust part is letting go. Letting it scaffold, refactor, write the boring stuff. Not hovering over every token.

The verify part is remembering that confident and correct aren't the same thing. It will absolutely write code that looks fine, runs fine, and does the wrong thing in ways you won't catch until production.

So you read it. You test it. You treat it like code from a very fast junior dev who never gets tired but also never asks clarifying questions, unless you tell it to ask.
